# إدارة أقسام الصفحة الرئيسية

## نظرة عامة
هذه الصفحة تسمح للمسؤولين بإدارة بيانات الأقسام المعروضة في الصفحة الرئيسية، بما في ذلك:
- **اسم القسم**: الاسم العربي الذي يظهر للمستخدمين
- **أيقونة القسم**: الصورة/الأيقونة المميزة للقسم

## الموقع
يمكن الوصول للصفحة من خلال:
- القائمة الجانبية: `الأقسام والتصنيفات` > `إدارة أقسام الصفحة الرئيسية`
- الرابط المباشر: `/category-homepage-management`

## المميزات

### 1. عرض جميع الأقسام
- عرض شبكي منظم لجميع الأقسام
- كل بطاقة قسم تحتوي على:
  - الأيقونة الحالية
  - اسم القسم
  - slug القسم (المعرّف)
  - زر "إدارة بيانات القسم"

### 2. البحث والتصفية
- شريط بحث سريع للعثور على قسم معين
- البحث يعمل على اسم القسم

### 3. تعديل بيانات القسم
عند النقر على "إدارة بيانات القسم"، تفتح نافذة منبثقة تحتوي على:

#### اسم القسم
- حقل نصي لتعديل اسم القسم
- يتم عرض الاسم الحالي كقيمة افتراضية

#### أيقونة القسم
- رفع أيقونة جديدة (صورة)
- معاينة فورية للأيقونة الجديدة قبل الحفظ
- عرض الأيقونة الحالية إذا لم يتم رفع أيقونة جديدة
- يدعم جميع صيغ الصور (PNG, JPG, JPEG, WebP)

### 4. حفظ التعديلات
- زر "حفظ التعديلات" لتطبيق التغييرات
- رسائل تأكيد عند النجاح
- رسائل خطأ واضحة عند الفشل
- تحديث فوري للبيانات بعد الحفظ

## API Endpoints المستخدمة

### 1. جلب جميع الأقسام
```
GET /api/admin/categories
```
**Response:**
```json
{
  "data": [
    {
      "id": 1,
      "name": "العقارات",
      "slug": "real_estate",
      "icon_url": "https://domain.com/storage/uploads/categories/icon.png",
      "is_active": 1,
      "sort_order": 1
    }
  ]
}
```

### 2. جلب بيانات قسم محدد
```
GET /api/admin/categories/{id}
```
**Response:**
```json
{
  "data": {
    "id": 1,
    "name": "العقارات",
    "icon_url": "...",
    "is_active": 1,
    "sort_order": 1
  }
}
```

### 3. تحديث بيانات القسم
```
POST /api/admin/categories/{id}
Content-Type: multipart/form-data
```
**Body:**
- `_method`: "PUT" (مطلوب)
- `name`: اسم القسم الجديد (اختياري)
- `icon`: ملف الصورة (اختياري)
- `is_active`: 1 أو 0 (اختياري)

**Response:**
```json
{
  "message": "تم تحديث القسم بنجاح",
  "data": {
    "id": 1,
    "name": "العقارات",
    "icon_url": "...",
    "is_active": 1
  }
}
```

### 4. تقرير الاستخدام (للرجوع المستقبلي)
```
GET /api/admin/categories/usage-report
```
**Response:**
```json
{
  "data": [
    {
      "id": 1,
      "name": "العقارات",
      "slug": "real_estate",
      "listings_count": 520,
      "icon_url": "..."
    }
  ]
}
```

## التقنيات المستخدمة

### Frontend
- **Next.js 14** App Router
- **TypeScript** للأمان والوضوح
- **Client Component** للتفاعل الديناميكي

### Services & Models
- `services/makes.ts`: 
  - `fetchAdminCategories()`: جلب جميع الأقسام
  - `fetchCategoryHomepage()`: جلب بيانات قسم محدد
  - `updateCategoryHomepage()`: تحديث بيانات القسم
  - `fetchCategoriesUsageReport()`: جلب تقرير الاستخدام

- `models/makes.ts`:
  - `CategoryHomepageItem`: نموذج بيانات القسم
  - `CategoryHomepageUpdateRequest`: نموذج طلب التحديث
  - `CategoryHomepageResponse`: نموذج استجابة API

## الأمان
- جميع الطلبات تتطلب **Bearer Token** للمصادقة
- يتم استخراج التوكن من `localStorage.getItem('authToken')`
- جميع الطلبات تحدد `Accept: application/json` لضمان استجابات JSON

## التصميم والواجهة
- تصميم عصري متجاوب مع جميع الشاشات
- بطاقات منظمة مع تأثيرات hover جذابة
- نافذة منبثقة (Modal) أنيقة للتعديل
- رسائل Toast للإشعارات
- معاينة فورية للصور
- تصميم RTL كامل

## الملفات ذات الصلة
- `/app/category-homepage-management/page.tsx` - صفحة Server Component
- `/app/category-homepage-management/CategoryHomepageClient.tsx` - المكون الرئيسي

- `/components/Sidebar.tsx` - القائمة الجانبية (تحتوي على الرابط)

## سيناريوهات الاستخدام

### 1. تغيير اسم قسم
1. ابحث عن القسم أو اعثر عليه في القائمة
2. اضغط على "إدارة بيانات القسم"
3. عدّل اسم القسم في الحقل النصي
4. اضغط "حفظ التعديلات"
5. سيتم تحديث الاسم فوراً في جميع أنحاء التطبيق

### 2. تغيير أيقونة قسم
1. اضغط على "إدارة بيانات القسم"
2. اضغط على "Choose File" في حقل الأيقونة
3. اختر صورة من جهازك
4. شاهد المعاينة الفورية
5. اضغط "حفظ التعديلات"
6. سيتم رفع الأيقونة وتحديثها

### 3. تحديث الاسم والأيقونة معاً
1. افتح نافذة التعديل
2. عدّل الاسم
3. ارفع أيقونة جديدة
4. احفظ - سيتم تحديث كليهما في طلب واحد

## ملاحظات تطويرية

### حالات الخطأ
- يتم عرض رسائل خطأ واضحة للمستخدم
- يتم تسجيل الأخطاء في Console للمطورين
- يتم التعامل مع فشل تحميل الصور بشكل صحيح

### الأداء
- تحميل البيانات مرة واحدة عند فتح الصفحة
- التحديث المحلي الفوري بعد الحفظ (بدون إعادة تحميل كامل)
- معاينة الصور باستخدام `URL.createObjectURL()` بدون رفع

### التوسعات المستقبلية
- يمكن إضافة خاصية السحب والإفلات لترتيب الأقسام
- يمكن إضافة تصفية حسب الحالة (نشط/غير نشط)
- يمكن إضافة صفحة تقرير الاستخدام باستخدام `/usage-report` endpoint
- يمكن إضافة خاصية الحذف (إذا سمح API)

## استكشاف الأخطاء

### المشكلة: لا تظهر الأيقونات
**الحل**: تحقق من:
- صحة رابط الأيقونة في استجابة API
- دعم المتصفح لصيغة الصورة
- وجود CORS للصور الخارجية

### المشكلة: فشل حفظ التحديثات
**الحل**: تحقق من:
- وجود Bearer Token صالح
- حجم الصورة المرفوعة (إذا كانت كبيرة جداً)
- اتصال الإنترنت
- صلاحيات المستخدم

### المشكلة: لا تظهر رسائل النجاح/الخطأ
**الحل**: تأكد من:
- عمل نظام Toast بشكل صحيح
- عدم وجود أخطاء JavaScript في Console
